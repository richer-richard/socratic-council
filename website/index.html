Local-first multi-agent desktop debate

Seven minds, one chamber, no server in the middle.

Socratic Council runs a seven-agent seminar on any topic. You bring your own provider keys, choose the room’s models, attach evidence, and watch the discussion unfold with search, quoting, reactions, export, conflict graphs, and cost tracking built directly into the desktop app.

Install and first run

There are two entry paths, and neither one asks you to trust a hosted black box.

Quick install

macOS can take the single-command route.

The repository ships a guided `install.sh` path that checks Xcode CLT, Homebrew, Node.js, pnpm, and Rust, then builds and places the app into `/Applications`. 

git clone https://github.com/richer-richard/socratic-council.git
cd socratic-council
./install.sh
Manual path

Windows, Linux, and careful macOS builders can inspect every step.

The README covers prerequisites, Node 22+, Rust, pnpm, the development loop, the production build, and the platform-specific bundle outputs.

  • Node.js 22 or newer
  • pnpm 9.15.0 via corepack
  • Rust stable 1.77.2 or newer
  • Tauri v2 system prerequisites by platform
First run setup

Bring your own provider keys, then tune the room.

Settings cover provider credentials, model selection, global proxy settings, and the moderator toggle. Keys are stored locally in the desktop app, not on a server you do not control.

Evidence surface

This is where the app becomes more useful than a plain transcript window.

Search, export, conflict visualization, and logs are built into the workflow instead of being treated as afterthoughts.

Search

Jump straight to the line that matters.

Transcript search resolves directly back into the timeline instead of a detached index.

Export

Leave with something you can actually share.

Markdown JSON PDF DOCX PPTX

The export pipeline is local and structured. It carries the transcript forward with summaries, charts, and computed statistics.

Conflict graph

Disagreement is rendered, not hidden.

Conflict relationships and pressure points are surfaced as a reviewable visual layer.

Oracle tool

Agents can request verification inside the room.

Tool results return into the transcript as normal messages, so evidence stays visible to the whole council instead of vanishing into hidden state.

Logs

When something breaks, the app leaves a trail.

Provider calls, app behavior, and error surfaces are easier to inspect because the product already assumes debugging is part of the real workflow.

For builders

A small monorepo, a desktop shell, and enough structure to evolve without theater.

Monorepo layout 
socratic-council/
apps/desktop        # Tauri v2 + React app
packages/shared     # types, constants, models
packages/sdk        # provider SDK and transport
packages/core       # bidding, conflict, memory
Core commands 
pnpm --filter @socratic-council/desktop tauri:dev
pnpm --filter @socratic-council/desktop tauri:build
pnpm test
pnpm lint
pnpm typecheck
