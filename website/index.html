Socratic Council

Eight agents enter the same chamber with your question, your files, and no hosted control plane between them.

Bring your own provider keys. Run the discussion locally. Search the transcript, track disagreement, verify evidence, and export the room when the argument is finished.

How the chamber moves

The product is not a chat skin. It is a discussion loop with memory, bidding, and visible evidence.

A turn begins with context building, passes through speaker selection and provider streaming, then folds tool results, quotes, reactions, and analytics back into the transcript surface.

01

Context is built from the live transcript, memory, attachments, and session state.

02

Each round chooses a speaker instead of flattening every reply into one assistant voice.

03

Structured actions stay visible in the room, so verification and follow-up are part of the record.

What leaves the room

Search, conflict, cost, and exports are not side quests after the debate.

The transcript remains searchable, tool results stay in context, disagreement gets rendered instead of buried, and the export layer writes structured artifacts directly to disk.

Search

Exports
Markdown, JSON, PDF, DOCX, and PPTX all come from the same local transcript.
Logs
Provider behavior, failures, and app events remain inspectable when things go wrong.

Build from source

The repo is the product perimeter, so the install path is explicit by design.

Quick install exists for macOS. The manual path remains the source-of-truth for people who want exact control over prerequisites, toolchains, and the desktop bundle.

Quick install

`install.sh` checks Xcode CLT, Homebrew, Node.js, pnpm, and Rust, then builds and places the app in `/Applications`. 

git clone https://github.com/richer-richard/socratic-council.git
cd socratic-council
./install.sh

Manual install

The README walks through system prerequisites, Node 22+, pnpm 9.15.0, Rust stable, and the Tauri development and production commands for macOS, Windows, and Linux.

  • Node.js 22 or newer
  • pnpm 9.15.0 via corepack
  • Rust stable 1.77.2 or newer
  • Tauri v2 platform prerequisites

For builders

The repo stays compact: one desktop shell, three shared packages, and a readable build path.

The monorepo is small enough to reason about without ceremony. Desktop UI lives in `apps/desktop`; orchestration, transport, and shared contracts live in the workspace packages behind it.

Core commands

pnpm --filter @socratic-council/desktop tauri:dev
pnpm --filter @socratic-council/desktop tauri:build
pnpm test
pnpm lint
pnpm typecheck
